Introduction to Java

What Java actually is — and why the JVM changes everything

Java is not just a language. It is a platform. Most programming languages compile to instructions specific to one operating system — your code for Windows won't run on Mac. Java broke this rule in 1995 by inventing a middle layer: the Java Virtual Machine (JVM).

You write Java code once. The compiler converts it to a universal format called bytecode. The JVM on any device reads that bytecode and executes it natively. This is the "Write Once, Run Anywhere" promise — and it is why Java still powers Android, banking systems, and enterprise software three decades later.

"The JVM is like a universal translator between your code and the machine."

⚡ Key Facts About Java
  • Object-Oriented: Java organizes code around objects — real-world things with properties and behaviors
  • Statically Typed: You must declare variable types before using them — this catches bugs at compile time
  • Garbage Collected: Java automatically frees unused memory — no manual memory management
  • Case Sensitive: HelloWorld and helloworld are completely different names in Java

Variables & Data Types

How Java names, stores, and types the data in your program

A variable is a named slot in your computer's RAM. When you declare int age = 22, Java does three things simultaneously: it reserves memory space, labels it age, and writes the number 22 into it.

The data type tells Java how much memory to reserve and how to interpret the bits stored there. A text character and a decimal number need completely different storage structures — this is why Java requires you to declare types upfront.

🚨 Common Beginner Trap

Local variables have no default value. If you declare int x; and try to print it before assigning a value, Java will refuse to compile. This is a safety feature — always initialize your variables.

// Declaring and initializing three common types
int    age    = 22;             // whole number → 4 bytes reserved in RAM
double salary = 45000.50;      // decimal → 8 bytes reserved in RAM
String name   = "Rahul";        // text → stored as object in heap memory

System.out.println("Age: "    + age);     // Age: 22

Control Flow — Making Decisions

Teaching your program to choose different paths

Real programs don't execute every line every time. They branch — they choose what to do based on conditions. This is control flow: the ability to direct your program down different paths depending on the data it encounters.

The if-else statement is the fundamental decision-making tool. Think of it as a fork in the road — your program evaluates a condition, and if it's true, it takes one path; otherwise it takes another.

"If the condition is true, do this. Otherwise, do that. That's 90% of programming logic."

💡 The Grading Formula

The marks check program from this lesson is a chained if-else. Java evaluates each condition top to bottom and executes the first one that is true — the rest are skipped. Order matters!

int marks = 75;

if (marks >= 50) {
    System.out.println("Student Passed");

JAVA — Grading system (exercise solution)
int marks = 82;
char grade;

if (marks >= 90) {
    grade = 'A';          // top scorers
} else if (marks >= 75) {
    grade = 'B';          // good scorers
} else if (marks >= 50) {
    grade = 'C';          // passing
} else {
    grade = 'F';          // failed
}

System.out.println("Grade: " + grade);
// Output: Grade: B

Loops — Mastering Repetition

How to make your program do something many times without repeating code

Imagine you need to print the numbers 1 to 1000. Writing 1000 println calls is absurd. Loops solve this by repeating a block of code automatically until a condition is no longer true.

The for loop is designed for situations where you know exactly how many times you want to repeat. It packs three things into one line: where to start, when to stop, and how to step forward.

"A loop is just a shortcut for 'do this again... and again... until I say stop.'"

🔄 The 3 Parts of a For Loop

for (init ; condition ; update)

  • int i = 1 — Start: run once before the loop begins
  • i <= 5 — Condition: checked before EACH iteration
  • i++ — Update: runs AFTER each iteration

Student.java — The Blueprint
class Student {

    // Instance variables — each student owns these
    int    id;
    String name;
    int    age;

    // Constructor — called when creating a student
    Student(int id, String name, int age) {
        this.id   = id;    // 'this' refers to the current object
        this.name = name;
        this.age  = age;
    }
}
Main.java — Using the Blueprint
ArrayList<Student> students = new ArrayList<>();

// ADD — create objects from the Student class
students.add(new Student(1, "Priya", 20));
students.add(new Student(2, "Arjun", 22));

// DISPLAY — loop over all students
for (Student s : students) {
    System.out.println(s.id + " | " + s.name + " | " + s.age);
}
// Output:
// 1 | Priya | 20
// 2 | Arjun | 22
Concepts this project teaches
OOP — Classes

Student is a class — a blueprint. Each student you create is an object built from that blueprint.

Collections — ArrayList

Unlike arrays, ArrayList can grow and shrink dynamically as you add or remove students.

For-each Loop

The for (Student s : students) syntax is a cleaner way to loop over collections.
